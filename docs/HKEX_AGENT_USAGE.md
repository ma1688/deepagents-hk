## 快速开始

### 启动 HKEX Agent

```bash
# 方式 1：直接运行（推荐）
hkex

# 方式 2：通过 Python 模块运行
python -m src.cli.main

# 方式 3：使用 uv 运行
uv run hkex
```

## 故障排查

### 问题 1：找不到公告

**症状**：
```
❌ 未找到符合条件的公告
```

**解决方案**：
1. 检查股票代码是否正确（应为 5 位数字，如 `00700`）
2. 检查日期范围是否合理
3. 尝试使用更宽泛的关键词（如 "报告" 代替 "中期报告"）

```bash
# 正确示例
HKEX Agent> 00700 最新报告

# 错误示例
HKEX Agent> 700 最新报告  # ❌ 缺少前导零
```

### 问题 2：PDF 下载失败

**症状**：
```
❌ PDF 下载失败: Connection timeout
```

**解决方案**：
1. 检查网络连接
2. 检查防火墙设置
3. 重试下载

```bash
# 重试下载
HKEX Agent> 重新下载 00700 最新中期报告
```

### 问题 3：Token 溢出

**症状**：
```
❌ Error code: 400 - number of input tokens (195354) have exceeded max_prompt_tokens (163840)
```

**解决方案**：
✅ **已自动修复**：系统会自动检测大型 PDF（> 50k 字符）并保存到缓存，不会再出现此问题。

如果仍然遇到问题，请检查：
1. 是否使用最新版本（`git pull origin master`）
2. 缓存目录是否可写（`pdf_cache/`）

### 问题 4：颜色空间警告

**症状**：
```
Cannot set gray stroke color because /'P0' is an invalid float value
Cannot set non-stroke color because 2 components are specified...
```

**解决方案**：
✅ **已自动修复**：系统会自动抑制这些警告，不影响功能。

## 性能优化

### 1. 缓存机制

**PDF 缓存**：
- 已下载的 PDF 会自动缓存到 `pdf_cache/{stock_code}/` 目录
- 再次查询相同公告时，直接使用缓存，无需重新下载

**提取内容缓存**：
- 大型 PDF（> 50k 字符）的提取内容会保存到 `.txt` 和 `_tables.json` 文件
- 后续分析可以直接读取缓存，无需重新提取

**清理缓存**：
```bash
# 清理 30 天前的缓存
HKEX Agent> 清理缓存

# 清理特定股票的缓存
HKEX Agent> 清理 00700 的缓存

# 清理所有缓存
HKEX Agent> 清理所有缓存
```

### 2. 并行处理

对于批量查询，系统会自动并行处理多个请求：

```bash
# 并行查询多个股票（自动优化）
HKEX Agent> 00700、00875、03800 最新公告摘要
```

### 3. 智能截断

大型 PDF（> 50k 字符）会自动触发智能截断：
- 返回前 5k 字符预览 + 前 5 个表格
- 完整内容保存到缓存文件
- LLM 可以按需读取完整内容

**查看截断状态**：
```bash
HKEX Agent> 查看 03800 2024年报的提取状态
```

**预期输出**：
```
📊 提取状态
- 文本长度: 206,345 字符
- 表格数量: 45 个
- 截断状态: ✅ 已截断
- 缓存路径:
  * 文本: pdf_cache/03800/2025-04-29-2024年报.txt
  * 表格: pdf_cache/03800/2025-04-29-2024年报_tables.json
```

## 配置选项

### 环境变量

在 `.env` 文件中配置：

```bash
# LLM 模型配置
SILICONFLOW_MODEL=deepseek-ai/DeepSeek-V3.1-Terminus
SILICONFLOW_PDF_MODEL=Qwen/Qwen2.5-7B-Instruct
SILICONFLOW_REPORT_MODEL=Qwen/Qwen2.5-72B-Instruct

# 温度参数（创造性 vs 确定性）
SILICONFLOW_TEMPERATURE=0.7           # 主 Agent（平衡）
SILICONFLOW_PDF_TEMPERATURE=0.5       # PDF 分析（更确定）
SILICONFLOW_REPORT_TEMPERATURE=0.7    # 报告生成（更创造）

# UI 配置
HKEX_ASCII_FONT=slant                 # 横幅字体
HKEX_RAINBOW=true                     # 彩虹效果
```

### 字体选择

支持 571 种 ASCII 艺术字体，常用推荐：

```bash
# 在 .env 中修改
HKEX_ASCII_FONT=slant      # 默认，斜体风格
HKEX_ASCII_FONT=banner     # 方块风格
HKEX_ASCII_FONT=big        # 大字体
HKEX_ASCII_FONT=block      # 块状风格
HKEX_ASCII_FONT=bubble     # 气泡风格
HKEX_ASCII_FONT=digital    # 数字风格
HKEX_ASCII_FONT=standard   # 标准风格
```

**预览效果**：

## 最佳实践

### 1. 查询优化

**推荐**：使用具体的公告类型
```bash
✅ HKEX Agent> 00700 最新中期报告
✅ HKEX Agent> 03800 2024年报
```

**不推荐**：使用模糊的关键词
```bash
❌ HKEX Agent> 00700 最新的那个东西
❌ HKEX Agent> 03800 最近的
```

**推荐**：一次性提交所有查询
```bash

```

**不推荐**：逐个查询
```bash
❌ HKEX Agent> 00700 最新公告摘要
❌ HKEX Agent> 00875 最新公告摘要
❌ HKEX Agent> 03800 最新公告摘要
```

### 3. 缓存利用

**推荐**：重复查询时利用缓存
```bash
# 第一次查询（会下载 PDF）
HKEX Agent> 00700 最新中期报告

# 第二次查询（使用缓存，速度快）
HKEX Agent> 00700 最新中期报告的财务数据
```

**推荐**：定期清理旧缓存
```bash
# 每月清理一次

```

## 常见问题 (FAQ)

### Q1: 如何查看支持的股票代码？

**A**: HKEX Agent 支持所有港交所上市公司的股票代码（5 位数字格式）。

```bash
# 示例
00700  # 腾讯控股
00875  # 中国金茂
03800  # 保利协鑫能源
```

### Q2: 如何指定查询日期？

**A**: 在查询中包含日期信息：

```bash
HKEX Agent> 00700 2025年8月的中期报告
HKEX Agent> 03800 2024年报
HKEX Agent> 00875 2025年9月12日的配售公告
```

### Q3: 如何导出分析结果？

**A**: 使用 "生成摘要md" 命令：

```bash
HKEX Agent> 00700 最新中期报告，并生成摘要md
```

结果会保存到 `md/` 目录下。

### Q4: 如何处理大型年报？

**A**: 系统会自动处理大型 PDF（> 50k 字符）：
- 自动保存到缓存
- 返回预览内容
- LLM 可以按需读取完整内容

无需手动干预。

### Q5: 如何自定义分析维度？

**A**: 在查询中明确指定分析维度：

```bash
HKEX Agent> 03800 2024年报的盈利能力和偿债能力分析

```
